# Книжный каталог (Laravel API)

API-приложение для управления каталогом книг с административной панелью на Filament, системой отзывов и логов активности.

## Требования
* PHP 8.4
* PostgreSQL 17
* Docker & Docker Compose

## Выполненные задачи

### Базовый функционал и API
* **Задачи 0-2**: Настройка Docker-окружения (php-fpm, nginx, pgsql), создание схемы БД (пользователи, авторы, книги, жанры).
* **Задачи 3-4**: Авторизация по одноразовому коду (выводится в логи), просмотр и редактирование профиля.
* **Задача 5**: API-методы для просмотра авторов, книг и жанров (с выводом количества книг и пагинацией).
* **Задача 6**: Фильтры книг по автору, жанру, диапазону годов издания. Сортировка по цене и дате создания.
* **Задача 7**: Раздел «Избранное» для пользователей (добавление, удаление, просмотр списка).
* **Задача 8**: Загрузка обложек книг и аватаров авторов через Spatie Media Library.
* **Задачи 9-10**: Добавление отзывов к книгам (оценки, комментарии, анонимность) и система лайков/дизлайков для отзывов.
* **Задача 11**: Локализация всех ответов API и ошибок валидации (русский/английский языки в lang/ru, lang/en).

### Панель администратора и логирование
* **Задачи 15-20**: Панель администратора на Filament PHP. Добавлены CRUD для администраторов, авторов, книг (с отзывами и обложками), жанров и пользователей.
* **Задача 21**: Система логирования активности (создание, обновление, удаление сущностей, вход и регистрация) на базе Eloquent Observers. Просмотр логов в админке.

### Фильтры и документация
* **Задача 23**: Добавлены новые характеристики книг (издательство, страницы, обложка, вес и т.д.). Реализован эндпоинт GET /api/books/filters для получения динамического списка доступных фильтров.
* **Задача 24**: Сортировка книг по количеству отзывов и году издания.
* **Задача 25**: API-документация через Scramble (/docs/api).

## Быстрый старт

1. Скопировать конфигурацию окружения:
   ```bash
   cp .env.example .env
   ```

2. Запустить контейнеры:
   ```bash
   docker compose up -d --build
   ```

3. Выполнить миграции и наполнить БД тестовыми данными:
   ```bash
   docker compose exec php-fpm php artisan migrate --seed
   ```

## Столкнулись с ошибками и их решения

1. **Конфликт очередности роутов (`routes/api.php`)**
   * **Проблема**: Роут `Route::get('/books/{book}', ...)` был объявлен *выше*, чем `Route::get('/books/filters', ...)`. Из-за этого запрос к `/api/books/filters` воспринимался Laravel как попытка загрузить книгу с ID `"filters"`, что приводило к ошибке 500 (недопустимый синтаксис bigint).
   * **Решение**: Переместили `/books/filters` выше роута `/books/{book}`.

2. **Проблема N+1 запросов при получении оценок отзывов (`Review::user_rating`)**
   * **Проблема**: При обращении к динамическому атрибуту `user_rating` для каждого отзыва выполнялся отдельный SQL-запрос к таблице оценок отзывов (`review_rates`), создавая десятки лишних запросов на страницу.
   * **Решение**: Определили отношение `currentUserRate` в модели `Review` и настроили жадную загрузку (eager loading) через `reviews.currentUserRate` во всех контроллерах (`CatalogController`, `FavoriteController`).

3. **Случайное удаление роутов `/genres` и `/authors` при мерже**
   * **Проблема**: В процессе реорганизации маршрутов были случайно стерты роуты жанров и авторов, из-за чего тесты стали падать с ошибкой 404.
   * **Решение**: Роуты были восстановлены в `routes/api.php`.
